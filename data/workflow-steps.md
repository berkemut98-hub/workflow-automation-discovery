# Daily Recruiter Outreach — Repeatable Workflow Steps

Source: Notion page "📬 Daily Recruiter Outreach Log" (entry of June 21, 2026).
Note: The page is an output log, not an explicit SOP, so these steps are
**inferred** from what each entry documents (find recruiter → research their
recent content → pitch matched roles → avoid repeats → log it).

| # | Repeatable step | Category |
|---|-----------------|----------|
| 1 | Source candidate recruiters in target regions (AU/NZ/Canada) — e.g. scanning directories like Australia's National AI Centre for AI-focused boutique agencies | Research/lookup |
| 2 | Find each recruiter's contact email address | Research/lookup |
| 3 | Find a recent, specific hook for each — their LinkedIn article, salary guide, or market report (e.g. "NSW 2026 Tech Market Report") | Research/lookup |
| 4 | Cross-check each name against the "previously contacted — do not repeat" list to avoid duplicates | Judgement (dedup against a list) |
| 5 | Decide which roles/regions to pitch each recruiter based on their specialty (AU ops, Toronto/Vancouver relocation, perm vs. contract) | Judgement |
| 6 | Write a personalised opener that references the recruiter's recent content | Judgement |
| 7 | Assemble the email into standard format (opener + pitch + signature) | Formatting |
| 8 | Send the email from Gmail | Copy-paste between tools |
| 9 | Record the entry in this Notion log — name, email, why, what was pitched | Manual data entry |
| 10 | Append the newly-contacted names to the "previously contacted" list | Manual data entry |

## Caveats
- The page header states it is "updated automatically each weekday by the
  scheduled outreach task," so several steps (7–10, possibly 3) may already be
  automated in the Cowork setup. This table describes the logical workflow, not
  necessarily what is still done by hand.
- Only one day of data was available, so it was not possible to confirm which
  steps recur every time versus one-offs.
